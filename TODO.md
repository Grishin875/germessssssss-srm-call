  Ключевая бизнес-логика

  1. Изделие и его маршрут. У Product три флага, которые собирают
  производственный маршрут из сегментов:
  - needs_smd → склад СМД → СМД-монтаж → AOI → гравировка
  - is_receiver → + прошивка
  - needs_assembly → склад РЭА → выдача → сборка РЭА → ОТК
  - всегда → склад готовой продукции

  2. Многоуровневый MRP (confirm_production_order, рекурсивный, глубина до 10):
  при подтверждении ПЗ резервирует материалы по BOM. На нехватку — решение
  make-vs-buy: если у компонента есть активный BOM → создаётся дочерний ПЗ на
  выпуск узла (и рекурсивно подтверждается); если нет → авто-заявка на закупку.

  3. Прохождение цеха (complete_stage): закрытие этапов с складскими движениями
  (трансферы СМД↔РЭА), а при закрытии всех этапов — финализация: списание
  зарезервированных материалов в производство + оприходование готовой продукции.

  4. Контроль качества и брак (QCInspection → ReworkTask →
  ReworkMaterialRequest): два гейта (AOI после СМД, ОТК после сборки). При браке
  создаётся задача на переделку, при необходимости — запрос материалов, и
  requeue_after_rework возвращает изделие на контроль (AOI-брак→AOI;
  ОТК-брак→сборка→ОТК).

  5. Синхронизация статусов: статус клиентского sales.Order автоматически
  подтягивается из статусов связанных производственных заказов.

  Это соответствует вашей сохранённой памяти о [[order-production-flow]] —
  12-этапном маршруте с ветвлениями брак/приёмник.
